Current Goal

BudgetSpin is moving from a local-only Docker app to a self-hosted production app on a personal server reached over Tailscale.

Desired production access path:

  • Mac resolves budget-spin.ts.local via /etc/hosts.
  • Traffic goes over Tailscale to the server.
  • Server routes HTTP(S) through Traefik.
  • BudgetSpin runs behind Traefik as a production Docker service.
  • Local machine remains the development environment.

Production Hosting Plan

Recommended shape:

  • Keep docker-compose.yml for local dev.
  • Add docker-compose.prod.yml for production.
  • Add .env.production.example with required variables.
  • Keep secrets out of git.
  • Use a persistent named Docker volume for production SQLite data.
  • Use Traefik labels on the app service.
  • Run app with npm run db:migrate && npm run build && npm run start or build a production image whose runtime command starts Next in production.
  • Set NODE_ENV=production.
  • Set stable BUDGETSPIN_AUTH_SECRET.
  • Set DATABASE_PATH=/app/data/budgetspin.sqlite.

Traefik assumptions to verify on the server:

  • Traefik Docker network name, for example traefik.
  • HTTP vs HTTPS entrypoint names, commonly web and websecure.
  • Whether TLS certs are needed for .ts.local. Since access is over Tailscale and /etc/hosts, plain HTTP behind Tailscale may be acceptable for a first private setup, but HTTPS is cleaner if Traefik is already configured.
  • Whether Traefik watches Docker labels on that network.

Example production compose outline:

services:
  app:
    build: .
    command: sh -c "npm run db:migrate && npm run build && npm run start"
    environment:
      DATABASE_PATH: /app/data/budgetspin.sqlite
      BUDGETSPIN_AUTH_SECRET: ${BUDGETSPIN_AUTH_SECRET}
      NODE_ENV: production
    volumes:
      - budgetspin-data:/app/data
    networks:
      - traefik
    labels:
      - traefik.enable=true
      - traefik.http.routers.budgetspin.rule=Host(`budget-spin.ts.local`)
      - traefik.http.routers.budgetspin.entrypoints=web
      - traefik.http.services.budgetspin.loadbalancer.server.port=3000
    restart: unless-stopped

volumes:
  budgetspin-data:

networks:
  traefik:
    external: true

If using HTTPS:

      - traefik.http.routers.budgetspin.entrypoints=websecure
      - traefik.http.routers.budgetspin.tls=true

Deployment Model

Recommended initial deployment process:

  1. Develop locally.
  2. Run tests locally in Docker.
  3. Commit changes.
  4. SSH to server over Tailscale.
  5. Pull latest repo on server.
  6. Run production compose build and recreate.
  7. Verify /login and dashboard through http://budget-spin.ts.local.

Example server commands:

cd /opt/budgetspin
git pull
docker compose -f docker-compose.prod.yml build --pull=false
docker compose -f docker-compose.prod.yml up -d --force-recreate
docker compose -f docker-compose.prod.yml logs --tail=100 app

Backup Model

SQLite is the production data source, so backups matter.

Recommended backup command from the server:

docker compose -f docker-compose.prod.yml exec -T app sqlite3 /app/data/budgetspin.sqlite ".backup '/app/data/backups/budgetspin-$(date +%F-%H%M%S).sqlite'"

If the image does not include sqlite3, either add it to the image or run a temporary SQLite container against the same volume.

Before destructive migrations or reset operations, take a SQLite backup.

Security Notes

  • This is private self-hosting over Tailscale, not public Internet hosting.
  • Still use a strong BUDGETSPIN_AUTH_SECRET; changing it invalidates sessions.
  • Use a strong password for mdaneri.
  • Do not commit .env.production.
  • If exposing beyond Tailscale later, add HTTPS, stricter headers, and consider 2FA.
