BudgetSpin Runbook

Assumptions

  • Docker and Docker Compose are installed and running.
  • The supported app path is this repository root.
  • Local development uses docker-compose.yml.
  • Production uses docker-compose.prod.yml.
  • The app listens on port 3000 inside the container.
  • App data is SQLite.
  • Local data is stored in the Docker volume budgetspin_budgetspin-data.
  • Production data is stored in the Docker volume budgetspin-prod-data.
  • BUDGETSPIN_AUTH_SECRET signs session cookies. Keep it stable between restarts; changing it logs out existing sessions.
  • Automatic FX lookup requires outbound HTTPS access from the app container to https://api.frankfurter.dev.

Local Development

Run:

docker compose up --build

Open:

http://localhost:3000

On first run after auth is enabled, open /login and create the password for user mdaneri.

Stop:

docker compose down

Restart:

docker compose restart app

Logs:

docker compose logs -f app

Run migrations:

docker compose run --rm app npm run db:migrate

Run tests:

docker compose run --rm app npm test

Typecheck:

docker compose run --rm app npm run typecheck

Production build check:

docker compose build

Reset local data:

docker compose down -v
docker compose up --build

Do not run the reset command if you need to preserve local finance data.

Production Topology

Expected request path:

Mac browser
  -> budget-spin.ts.local
  -> /etc/hosts resolves to the server Tailscale IP
  -> Traefik on the server
  -> Docker router Host(`budget-spin.ts.local`)
  -> BudgetSpin app container on Docker network proxy

Production assumptions encoded in docker-compose.prod.yml:

  • Traefik Docker network defaults to proxy.
  • Traefik entrypoint defaults to websecure.
  • TLS defaults to enabled.
  • Hostname defaults to budget-spin.ts.local.
  • No host port is published by the app container.

Adjust those values in .env.production on the server if needed.

Production First Setup

On the server, clone or copy this repo to the target directory. The examples below use /opt/budgetspin.

cd /opt/budgetspin
cp .env.production.example .env.production

Edit .env.production and set a long random value:

BUDGETSPIN_AUTH_SECRET=replace-with-a-long-random-secret

If the Traefik Docker network is not proxy, set:

TRAEFIK_DOCKER_NETWORK=actual-network-name

If Traefik is not using websecure or does not terminate TLS for this private hostname, set the matching values:

TRAEFIK_ENTRYPOINT=web
TRAEFIK_TLS=false

Confirm the Traefik network exists:

docker network inspect proxy

Build and start production:

docker compose --env-file .env.production -f docker-compose.prod.yml build --pull=false
docker compose --env-file .env.production -f docker-compose.prod.yml up -d

Check status:

docker compose --env-file .env.production -f docker-compose.prod.yml ps
docker compose --env-file .env.production -f docker-compose.prod.yml logs --tail=100 app

Mac Hostname

Add the server Tailscale IP to /etc/hosts on the Mac:

100.x.y.z budget-spin.ts.local

Then verify resolution and route from the Mac:

curl -k -I https://budget-spin.ts.local/login

If TRAEFIK_TLS=false, use:

curl -I http://budget-spin.ts.local/login

Deployments

Normal flow:

  1. Develop locally.
  2. Run typecheck/tests locally.
  3. Commit changes.
  4. Push or otherwise make the commit available on the server.
  5. SSH to the server over Tailscale.
  6. Pull latest code.
  7. Rebuild and recreate the production container.
  8. Verify logs and /login.

This repo must have commits and a git remote before the git pull based flow works. If the server is bootstrapped by copying files instead, keep .env.production and the Docker volume on the server and do not overwrite them from the Mac.

Manual deployment on the server:

cd /opt/budgetspin
git pull --ff-only
docker compose --env-file .env.production -f docker-compose.prod.yml build --pull=false
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --remove-orphans
docker compose --env-file .env.production -f docker-compose.prod.yml logs --tail=100 app

Remote deployment from the Mac:

BUDGETSPIN_SSH_TARGET=user@server ./scripts/deploy-prod.sh

Optional deployment variables:

BUDGETSPIN_REMOTE_DIR=/opt/budgetspin
BUDGETSPIN_BRANCH=main

Rollback

Rollback assumes the previous commit is still available in git.

cd /opt/budgetspin
git log --oneline -5
git checkout <known-good-commit>
docker compose --env-file .env.production -f docker-compose.prod.yml build --pull=false
docker compose --env-file .env.production -f docker-compose.prod.yml up -d --remove-orphans

After rollback, verify:

docker compose --env-file .env.production -f docker-compose.prod.yml ps
docker compose --env-file .env.production -f docker-compose.prod.yml logs --tail=100 app

Production Data

Before destructive operations or risky migrations, create a SQLite backup from inside the app container. This uses SQLite VACUUM INTO through Node’s node:sqlite module, so it does not require the sqlite3 CLI in the image.

BACKUP="budgetspin-backup-$(date +%F-%H%M%S).sqlite"
docker compose --env-file .env.production -f docker-compose.prod.yml exec -T app node --input-type=module -e "import { DatabaseSync } from 'node:sqlite'; const db = new DatabaseSync('/app/data/budgetspin.sqlite'); db.exec(\"VACUUM INTO '/app/data/$BACKUP'\"); db.close();"
docker compose --env-file .env.production -f docker-compose.prod.yml cp "app:/app/data/$BACKUP" "./$BACKUP"

To restore from that file, stop the app first:

docker compose --env-file .env.production -f docker-compose.prod.yml down
docker compose --env-file .env.production -f docker-compose.prod.yml run --rm --no-deps app sh -c 'rm -f /app/data/budgetspin.sqlite /app/data/budgetspin.sqlite-wal /app/data/budgetspin.sqlite-shm'
docker compose --env-file .env.production -f docker-compose.prod.yml create app
docker compose --env-file .env.production -f docker-compose.prod.yml cp ./budgetspin-backup.sqlite app:/app/data/budgetspin.sqlite
docker compose --env-file .env.production -f docker-compose.prod.yml up -d

Troubleshooting

Check production config rendering:

docker compose --env-file .env.production -f docker-compose.prod.yml config

Check Traefik can see labels:

docker compose --env-file .env.production -f docker-compose.prod.yml ps
docker inspect budgetspin-app-1 --format '{{json .Config.Labels}}'

Check the app directly from the server container namespace:

docker compose --env-file .env.production -f docker-compose.prod.yml exec app node -e "fetch('http://127.0.0.1:3000/login').then(async r => { console.log(r.status); process.exit(r.status < 500 ? 0 : 1); }).catch(error => { console.error(error); process.exit(1); })"

Common failures:

  • network proxy declared as external, but could not be found: set TRAEFIK_DOCKER_NETWORK to the server’s actual Traefik network or create the proxy network.
  • BUDGETSPIN_AUTH_SECRET missing: create .env.production from the example and set a stable random secret.
  • Browser cannot resolve budget-spin.ts.local: update /etc/hosts on the Mac with the server Tailscale IP.
  • Browser resolves but Traefik returns 404: verify hostname, entrypoint, TLS, labels, and Docker network.
